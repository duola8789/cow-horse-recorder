# 历史打卡记录编辑功能

## 概述

用户需要能够编辑历史打卡记录，以便：
- 补录忘记打卡的时间
- 修正打卡时间错误
- 设置请假状态

当前统计页面的详情弹窗有"编辑"按钮，但处于禁用状态，显示"编辑功能即将上线"。

## 目标

- 允许用户编辑当月的打卡记录
- 支持修改上班时间、下班时间
- 支持设置请假状态
- 提供友好的时间选择交互

## 范围

### 包含

- 新建 `updateRecord` 云函数
- 修改详情弹窗为可编辑模式
- 添加时间选择器组件
- 添加请假/正常状态切换
- 数据校验（时间格式、合理性）
- 保存成功后刷新列表数据

### 不包含

- 跨月编辑（只能编辑当前查看的月份）
- 编辑操作日志/审计记录
- 批量编辑功能
- 休息日打卡（休息日不可编辑）

## 成功标准

- [ ] 点击详情弹窗的"编辑"按钮，进入编辑模式
- [ ] 可以使用时间选择器修改上班/下班时间
- [ ] 可以切换正常/请假状态
- [ ] 选择请假后，时间输入隐藏
- [ ] 保存时进行数据校验（上班时间 < 下班时间）
- [ ] 保存成功后关闭弹窗，列表数据刷新
- [ ] 可以编辑当月任意工作日（包括今天）

## 风险与依赖

### 风险

- **低风险**: 编辑后需要清除本地缓存以确保数据一致性
- **低风险**: 时间选择器在不同小程序基础库版本的兼容性

### 依赖

- 微信小程序 `picker` 组件（内置，无额外依赖）
