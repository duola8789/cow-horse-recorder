# v1.1.0 优化

## 概述

1.0.0 版本上线后，收集到以下体验问题需要优化：

1. **首页冷启动无反馈** - 云函数启动期间页面无 loading，用户体验差
2. **打卡页面缺少当日时长** - 用户无法直观看到今天已工作多长时间
3. **统计页面时区错误** - 显示时间比实际打卡时间少 8 小时（UTC vs 北京时间）
4. **统计页面显示未来日期** - 当月未来的日期也展示出来，无意义且显得混乱
5. **统计页面当天无时长** - "进行中"的当天没有计算工作时长

## 目标

- 提升首屏加载体验
- 让用户实时了解当日工作时长
- 修复时区显示 bug
- 优化统计页面数据展示

## 范围

### 包含

- 打卡页面（`/pages/index`）loading 状态优化
- 打卡页面新增"今日已工作"时长展示
- 云函数 `getMonthlyRecords` 时区修复
- 云函数 `getMonthlyRecords` 只返回到当天的数据
- 云函数 `getMonthlyRecords` 计算当天进行中的工作时长

### 不包含

- 其他页面的 loading 优化（统计页面已有 loading）
- 打卡记录编辑功能（后续版本）
- 请假功能（后续版本）

## 成功标准

- [x] 首页加载时，数据区域显示 loading，直到数据加载完成
- [x] 打卡页面显示"今日已工作 X 小时 X 分"，每分钟更新
- [x] 统计页面显示的时间与实际打卡时间一致（北京时间）
- [x] 统计页面只展示到当天，不显示未来日期
- [x] 统计页面"进行中"的当天显示正确的工作时长

## 风险与依赖

### 风险

- **低风险**: 时区修复可能影响历史数据展示，需验证
- **低风险**: 时长计算逻辑需要与打卡页面和统计页面保持一致

### 依赖

- 无外部依赖
