# Change Proposal: UX 优化与完善

## Summary

完善牛马工时记录器的用户体验，包括登录流程优化、节假日数据同步机制、首页 UI 重新设计。

## Motivation

项目初始化后存在以下问题：

1. **登录流程不完善**：每个页面都调用 login，没有全局状态管理
2. **节假日同步缺失**：syncHolidays 云函数存在但没有定时触发器，首次上线也没有初始化机制
3. **首页 UI 太简陋**：字体小、按钮不醒目、缺少当前时间显示、交互不便捷

## Scope

### In Scope

- 登录流程重构：app.tsx 统一登录，全局状态管理
- 节假日同步：添加定时触发器配置，login 中添加兜底检测
- 首页 UI 重新设计：
  - 大字体实时时钟
  - 日期与工作日/休息日状态展示
  - Switch 切换上下班模式
  - 渐变彩色大圆形打卡按钮
  - 优化打卡状态卡片展示

### Out of Scope

- 统计页面和设置页面的 UI 优化（后续迭代）
- 用户信息授权（不需要头像/昵称）
- Tab 图标（已单独处理完成）

## Approach

### 1. 登录流程优化

- 在 `app.tsx` 的 `onLaunch` 中执行一次登录
- 使用 `Taro.getApp().globalData` 存储登录状态和用户信息
- 各页面通过全局状态判断登录情况，无需重复调用 login

### 2. 节假日同步机制

- 添加 `cloudfunctions/syncHolidays/config.json` 配置定时触发器
- 定时规则：每年 12 月 31 日 00:00 同步下一年数据
- 在 login 云函数中检测当年 holidays 数据是否存在，不存在则触发同步

### 3. 首页 UI 重设计

```
┌─────────────────────────────────────────┐
│                                         │
│          2026年3月19日 周四              │
│              [工作日]                    │
│                                         │
│              14:26:18                   │  ← 实时时钟 48px
│                                         │
│      上班 ○──────────● 下班              │  ← Switch 组件
│                                         │
│           ╭─────────────╮               │
│          │              │               │
│          │   🌅         │               │  ← 大圆按钮 180px
│          │  下班打卡    │               │     橙→红渐变
│          │              │               │
│           ╰─────────────╯               │
│                                         │
│    ┌──────────┬──────────┐              │
│    │ ☀️ 上班  │ 🌙 下班  │              │
│    │  09:32  │  --:--   │              │  ← 状态卡片
│    │ 已打卡 ✓ │  未打卡  │              │
│    └──────────┴──────────┘              │
│                                         │
└─────────────────────────────────────────┘
```

**配色方案**：

- 上班模式：蓝紫渐变 `#667eea → #764ba2`
- 下班模式：粉红渐变 `#f093fb → #f5576c`

## Success Criteria

- [ ] 登录只在 app 启动时执行一次
- [ ] 节假日定时触发器配置正确
- [ ] 首次使用时自动同步当年节假日数据
- [ ] 首页展示实时时钟（每秒更新）
- [ ] 首页展示当前日期和工作日状态
- [ ] Switch 可正确切换上下班模式
- [ ] 打卡按钮样式美观，交互流畅
- [ ] 打卡成功有震动反馈

## Risks

| 风险                               | 影响             | 缓解措施                                    |
| ---------------------------------- | ---------------- | ------------------------------------------- |
| 定时触发器首次部署可能需要手动配置 | 首年数据可能缺失 | login 兜底检测                              |
| 实时时钟频繁更新可能影响性能       | 耗电             | 使用 setInterval 而非 requestAnimationFrame |
| Switch 组件样式可能与 VantUI 冲突  | 样式不一致       | 自定义 Switch 组件                          |
